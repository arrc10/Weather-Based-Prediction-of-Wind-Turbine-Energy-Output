## Setup Instructions

### Step 1 – Install dependencies

```bash
pip install -r requirements.txt
```

### Step 2 – Generate the dataset

```bash
python generate_dataset.py
```

This creates `data/T1.csv` with ~105,000 rows of synthetic wind SCADA data.

> **Alternatively**: Download the real T1.csv from Kaggle:
> https://www.kaggle.com/datasets/berkerisen/wind-turbine-scada-dataset

### Step 3 – Train the model

```bash
python wind_turbine_energy_prediction.py
```

This:
- Preprocesses the dataset
- Generates EDA plots in `/plots/`
- Trains a Random Forest Regressor (R² ≈ 0.96+)
- Saves `Flask/power_prediction.sav`

### Step 4 – Add your OpenWeatherMap API key

1. Sign up at https://openweathermap.org/api
2. Copy your API key
3. Open `Flask/windApp.py` and replace:
   ```python
   OWM_API_KEY = "YOUR_OPENWEATHERMAP_API_KEY"
   ```
   with your actual key.

### Step 5 – Run the Flask app

```bash
cd Flask
python windApp.py
```

Open your browser at: **http://127.0.0.1:5000**

### Step 6 – (Optional) Test the model

```bash
python test_model.py
```

## Usage

1. **Weather Check**: Enter a city name (e.g., `Hyderabad`) or a 6-digit Indian pincode (e.g., `500072`) and click **Check**. Wind speed is auto-filled into the prediction form.
2. **Predict**: Enter Theoretical Power (KWh) and Wind Speed (m/s), then click **Generate Prediction**. The estimated energy output appears in the olive green result box.

---

## API Endpoints

| Method | Route      | Body (JSON)                              | Returns                     |
|--------|------------|------------------------------------------|-----------------------------|
| POST   | /weather   | `{ "query": "Hyderabad" }`                 | temp, humidity, pressure, wind_speed |
| POST   | /predict   | `{ "theoretical_power": 1500, "wind_speed": 4.12 }` | `{ "prediction": 1380.97 }` |

---

## Model Details

- **Algorithm**: Random Forest Regressor
- **Features**: Theoretical Power Curve (KWh), Wind Speed (m/s)
- **Target**: Active Power (kW)
- **Split**: 80% train / 20% validation
- **Key params**: `n_estimators=750, max_depth=4, max_leaf_nodes=500`

---

## Notes

- The wind direction column is dropped after EDA (near-zero correlation with power output).
- A free OpenWeatherMap API key activates within 2 hours of registration.
- For production deployment, use Gunicorn or uWSGI instead of Flask's dev server.
